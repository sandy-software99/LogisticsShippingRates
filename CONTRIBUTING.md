Contribution guidelines
Welcome Contributors!
Thank you for considering contributing to the centralised repository. This document outlines the guidelines for contributing to the devlopment of Shipping Rates and Calculations.

Code stye
Please follow the coding style and conventions used in the existing codebase. This helps maintain consistency across the project.

Documentation
Ensure that contributions are well-documented. Include comments in your code where necessary and provide a clear and concise description of your changes in the pull request.

Testing
Before submitting a pull request, make sure your changes have been tested thoroughly. Include relevant test cases and ensure that existing tests pass.

Issue Tracker
Check the issue tracker for any open issues or feature requests. If you're working on something, please comment on the issue to let others know.

Code review
All contributions will go through a code review process. Be open to feedback and and be willing to make changes if necessary. Code reviews help maintain code quality and consistency.

Thank you for your contribution!
